## Scoring Legend

| Symbol | Meaning |
|--------|---------|
| `+` | Positive-keyed item (higher agreement = higher trait score) |
| `-` | Reverse-keyed item (higher agreement = LOWER trait score; invert before scoring) |
| `W:` | Scoring weight for that dimension (1.0 = standard, 1.25 = high-signal) |

## Phase 1: RIASEC Interests

**Format:** "How much would you enjoy..." with 3-point response  
**Response scale:** Like (2) / Neutral (1) / Dislike (0)  
**Stem:** "How much would you enjoy each of these activities?"  
**Scoring:** Sum per type. Max = 8. Top 2-3 types form the Holland code.  
**Bias note:** Activity-based phrasing per O*NET Interest Profiler methodology. Gender-neutral language throughout. No occupational titles in stems to avoid stereotype activation.

## Appendix B: Scoring Pipeline Overview

```
Phase 1 (RIASEC)     --> Holland code (top 2-3 types)
Phase 2 (Big Five)    --> Personality profile (5 normalized scores)
Phase 3 (Values)      --> Schwartz quadrant scores + SDT need ranking
Phase 4 (Work DNA)    --> Work style composite (8 dimensions)
Phase 5 (Aptitudes)   --> Aptitude profile (8 self-assessed scores)
Phase 6 (Context)     --> Modifier layer (adjusts all above)

Combined --> career-brain.ts scoring engine:
  1. Base match score = weighted overlap of RIASEC + personality + values against career profiles
  2. Aptitude bonus/penalty for role-critical skills
  3. Work DNA alignment score
  4. Experience modifier (from experience-modifiers.ts)
  5. Gap pattern detection (from gap-patterns.ts)
  6. Combination rules overlay (from combination-rules.ts)
  7. Context modifiers (timeline, financial, learning style)
  8. Final ranked list of career matches with confidence scores
```

## Appendix C: Implementation Notes

### Presentation Order
- Phases 1-5 should be presented in order (builds engagement from easy to reflective).
- Within each phase, question order should be randomized to prevent order effects.
- For forced-choice pairs (Phase 3A), left/right option position should be randomized per user.

### Progress and UX
- Show phase progress bar (6 phases) and question progress within phase.
- Allow backward navigation to any completed phase.
- Save progress after each phase completion (resume on return).
- Estimated time remaining displayed per phase.

### Validation
- Phase 1: All 24 questions required. No skipping.
- Phase 2: All 20 questions required. No skipping.
- Phase 3: All 8 forced-choice pairs required. Ranking required. SDT questions required.
- Phase 4: All 10 scenarios required.
- Phase 5: All 8 aptitude self-assessments required.
- Phase 6: lc_stage, lc_experience, lc_skills (min 2), lc_domains (min 1), lc_learning (min 1) required. lc_age, lc_decision, lc_financial are optional/skippable.

### Data Storage
- Store raw answers (all question IDs and selected values) in the `raw_answers` JSON column.
- Store computed dimension scores in a new `dimension_scores` JSON column.
- Existing `strengths`, `values`, `personality_type` columns can be derived from dimension scores.
- Assessment version number should be stored to handle scoring changes over time.

### Migration from v1 (19 questions) to v2 (82 questions)
- Users who completed v1 keep their results.
- v2 is offered as "retake" with messaging: "Our assessment has been upgraded. Retake it for more accurate career matches."
- v1 results are not comparable to v2. Do not blend scores across versions.
